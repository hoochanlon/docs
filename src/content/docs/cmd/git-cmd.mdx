## push

```shell
git init
git remote -v
git remote remove origin
git remote add origin https://github.com/hoochanlon/aitii-tekisuto.git
git checkout -b main
git add .
git commit -m "123"
git push -f origin main
```

## reset


```shell
# 1. 先查看历史，确定要回退的版本
git log --oneline

# 2. 如果需要保留当前修改，先暂存
git stash
git stash list

# 3. 进行回退操作
git reset --hard <target_commit>

# 4. 如果需要恢复之前暂存的修改
git stash pop
```


```
# 撤销到合并前的提交，保留工作区更改
# 不小心 merge 了，撤回。
git reset --merge ORIG_HEAD
```


```
# 回到上一个提交
git reset --hard HEAD~1
# 或
git reset --hard HEAD^
```


## restore


```shell
# git restore [文件路径1] [文件路径2] ...
# git restore ：Git 用于恢复工作区文件状态的核心命令，用于撤销未提交的修改
git restore css/tutorial-content.css css/tutorial-toc.css
```

## pull


```shell
git pull origin main
```


## clone

:::tip
核心目的：最小化下载。
:::


```shell
# 1. 克隆仓库元数据，不下载文件内容
git clone --filter=blob:none --no-checkout https://github.com/hoochanlon/picx-images-hosting.git

# 2. 进入仓库目录
cd picx-images-hosting

# 3. 设置稀疏检出规则：包含所有文件，排除imgs目录
git sparse-checkout set --no-cone '/*' '!/imgs/*'

# 4. 检出主分支（根据实际分支名选择main或master）
git checkout main
```

浅克隆

```shell
# 下载最近1次提交
git clone --depth 1 <仓库URL>

# 浅克隆+单分支（更高效）
git clone --depth 1 --single-branch <仓库URL>
```
