:::tip
**重要提示**：本文档中的所有组件示例都需要在 `.mdx` 文件中使用，`.md` 文件不支持 JSX 组件。所有 `import` 语句必须放在文件顶部。

这套符号体系高度依赖特定平台的渲染支持，一旦切换平台便会失效。相当于是"软锁定"增加迁移成本。另外说一句：当你阅读到这篇内容时，Astro-Starlight 的短板也就显露出来了，就是当目录过长溢出整页时，不能做到自动滚动目录。


## 1. 盘白（Asides）

盘白用于突出显示重要信息，包括提示、警告、注意事项等。

### 代码示例

```mdx
:::note
这是一个提示框，用于显示一般性信息。
:::

:::tip
这是一个技巧提示框，用于显示有用的技巧或建议。
:::

:::caution
这是一个警告框，用于显示需要注意的事项。
:::

:::danger
这是一个危险警告框，用于显示可能导致严重后果的操作。
:::
```

### 实际效果

:::note
这是一个提示框，用于显示一般性信息。
:::

:::tip
这是一个技巧提示框，用于显示有用的技巧或建议。
:::

:::caution
这是一个警告框，用于显示需要注意的事项。
:::

:::danger
这是一个危险警告框，用于显示可能导致严重后果的操作。
:::


### 组件完整导入

以下是一个完整的 `.mdx` 文件模板，包含所有组件的导入：

## 2. 徽章（Badges）

徽章用于标记状态、版本或重要标签。

### 代码示例

```mdx
import { Badge } from '@astrojs/starlight/components'

- <Badge text="新功能" variant="tip" size="small" />
- <Badge text="已弃用" variant="caution" size="medium" />
- <Badge text="演示" variant="note" size="large" />
- <Badge text="200" variant="success" />
- <Badge text="实验性" variant="danger" />
```

**参数说明**

- `text`: 徽章显示的文本
- `variant`: 样式变体，可选值：`tip`、`caution`、`note`、`success`、`danger`
- `size`: 尺寸，可选值：`small`、`medium`、`large`

### 实际效果

- <Badge text="新功能" variant="tip" size="small" />
- <Badge text="已弃用" variant="caution" size="medium" />
- <Badge text="演示" variant="note" size="large" />
- <Badge text="200" variant="success" />
- <Badge text="实验性" variant="danger" />

## 3. 卡片（Cards）

卡片用于组织和展示相关内容。

### 单个卡片

#### 代码示例

```mdx
import { Card } from '@astrojs/starlight/components'

<Card title="卡片标题">
  这是卡片的内容，可以使用 **任何** Markdown 语法。
</Card>
```

#### 实际效果

<Card title="卡片标题">
  这是卡片的内容，可以使用 **任何** Markdown 语法。
</Card>

### 卡片网格

#### 代码示例

```mdx
import { CardGrid, Card } from '@astrojs/starlight/components'

<CardGrid>
  <Card title="卡片 1" icon="star">
    第一个卡片的内容。
  </Card>
  <Card title="卡片 2" icon="moon">
    第二个卡片的内容。
  </Card>
  <Card title="卡片 3" icon="rocket">
    第三个卡片的内容。
  </Card>
</CardGrid>
```

#### 实际效果

<CardGrid>
  <Card title="卡片 1" icon="star">
    第一个卡片的内容。
  </Card>
  <Card title="卡片 2" icon="moon">
    第二个卡片的内容。
  </Card>
  <Card title="卡片 3" icon="rocket">
    第三个卡片的内容。
  </Card>
</CardGrid>

## 4. 图标（Icons）

图标用于增强视觉效果。

### 代码示例

```mdx
import { Icon } from '@astrojs/starlight/components'

<Icon name="seti:lock" />
<Icon name="rocket" />
<Icon name="moon" />
```

### 实际效果

<Icon name="seti:lock" /> 锁定图标
<Icon name="rocket" /> 火箭图标
<Icon name="moon" /> 月亮图标

:::tip
图标名称可以使用 Starlight 内置图标或 Seti UI 图标集。更多图标请参考 [Starlight 图标文档](https://starlight.astro.build/components/icon/)。
:::



## 5. 步骤（Steps）

步骤组件用于创建分步骤的指南。

### 代码示例

```mdx
import { Steps } from '@astrojs/starlight/components'

<Steps>

1. 第一步：安装依赖

   \`\`\`bash
   npm install astro
   \`\`\`

2. 第二步：创建配置文件

   \`\`\`bash
   touch astro.config.mjs
   \`\`\`

3. 第三步：启动开发服务器

   \`\`\`bash
   npm run dev
   \`\`\`

</Steps>
```

### 实际效果

<Steps>

1. 第一步：安装依赖

   ```bash
   npm install astro
   ```

2. 第二步：创建配置文件

   ```bash
   touch astro.config.mjs
   ```

3. 第三步：启动开发服务器

   ```bash
   npm run dev
   ```

</Steps>

## 7. 文件树（File Tree）

文件树用于展示项目结构。

### 代码示例

```mdx
import { FileTree } from '@astrojs/starlight/components'

<FileTree>

- astro.config.mjs **重要配置文件**
- package.json
- README.md
- src
  - components
    - **Header.astro**
    - Footer.astro
  - content
    - docs
      - index.mdx
- public
  - favicon.svg

</FileTree>
```
