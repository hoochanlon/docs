---
title: 技术社区与项目推广的冷思考：从 52破解、Linux.do 到 GitHub Stars 的泡沫
---

在技术圈领域，linux.do 和 52 破解（以下简称“52”）的用户群体较为接近，但侧重点有所不同。52 更倾向于软件逆向、破解等话题，但审核很严，近乎于“破解内容无破解”，以及软件不能留有个人站点、联系方式，只能有论坛的地址，一不小心容易吃警告和违规，这样一看几近于无推广，所以不推荐。优点就是墙内可看到，SEO权重较高。

linux.do 相对领域宽泛没有很明显的侧重，nodeseek 也差不多，感觉 nodeseek 偏生活一点，linux.do 内容审核速度没 nodeseek 快，内容限制上反倒是 nodeseek 更严格一些，比如关于VPN代理方面。技术方面，在 linux.do 上推广受关注程度会更高。缺点就是这两平台都是墙外的。

投稿技术于大小博主，如：阮一峰、其他 GitHub 项目刊物作者；相关平台，如：HelloGihub；他们的月刊、周刊取决于其自身的流量大小，推广有的是注册，有的是提 issue 让对方收录，以及部分是 Merge 推广项目。BTW，所以每天看到了很多新奇的项目，解决了所谓的相关创造性的“需求”，但实际上你几乎是用不着的，和 App Store一样，创造需求场景讲故事，吸引你下载使用，体验后基本上就来到了后续的付费内容了。

又或是成本巨高的主动创造“流量”，在QQ群花钱购买粉丝、Stars 数，以及成本相对较低互关互粉等方式，来提升项目的曝光度和影响力，但这些方式的效果往往有限，甚至一眼假的。从学校毕业，初入社会面的是一家培训机构，在哪个时候我就有2-3k以上的 Stars 了，为了证明自身能力也谈及到这一点，但通过面试官的交谈中，可以说是明白的告诉我：他们对 GitHub Star follows 是很不屑一顾的，因为其自身是这产业的一环。
