---
title: 从脚本到产品：我的跨平台开发之旅
---

那个时候还没有 AI 出现，只能靠自己理解摸索编写代码。想出一个点子想要实现的话，绞尽脑汁榨干所有脑力来一点点实现点子中一小部分功能，几乎很少一次编写就通过编译的，拼命排查报错、调试 bug，很是心力俱疲。由于在这过程中耗费了大量精力，几乎也没有时间留给我总结了，像我这类天赋不佳的，真是纯靠经验编写代码和排错，一点都形成不了自己的体系。

说到这里，回想自己当年一昧听信网上及线下的编程培训课程，花了大价钱报班学习，结果学到的东西大多是些皮毛，根本没有系统地掌握编程的核心知识和技能。那些培训班的老师大多也只是些半吊子，根本没有实际项目经验不少都是包装出来的，企业也不傻，这样教出来的学生在当年也是几乎找不到工作的，更何况现在。

工作了两年后，在这期间不时写一些自动化脚本，虽对一些代码基础语法略知皮毛，甚至语言不通，也能对功能有自己的理解、构思、实现方式。后来 AI 出现了，并在 AI 辅助下完成了当初不可能完成的任务，如：批量处理文件操作、自动化爬虫处理数据生成报表。并在之后，我通过不断自学、参与开源项目，向有经验的开发者请教，实现了现代化网页产品的设计与开发，以及跨平台开发桌面应用。以下是我在重构 [Free NTFS for Mac](https://github.com/hoochanlon/Free-NTFS-for-Mac)、[Network Calculator](https://github.com/hoochanlon/network-calculator) 等应用 UI 界面了解到的信息。

通过与一位独立开发者的交流，我了解到：

<dd>在后端服务开发方面，Go 的性能表现更优秀。不过，对于日常用 Python 编写一些小工具或脚本来说，完全没有性能瓶颈，根本不需要过于纠结语言选择。</dd><br/>
    
<dd>至于桌面软件开发，Python 和 Go 都不是这方面的专业选择。它们确实也能实现，但并非强项，虽然可以通过某些库做出不错的用户界面，但这并不是它的优势所在。如果要做桌面应用，通常会选择 Visual Studio 配合 C++ 或 C# 来开发，或者使用 Electron、Flutter 等跨平台框架。</dd><br/>
    
<dd>像“拾光”这样的产品，涵盖了桌面客户端、网页版、小程序版、后端服务以及数据处理等多个环节，因此自然会涉及多种编程语言。</dd><br/>

另外，从一位拥有 7 年实际开发经验的程序员那里了解到：

<dd>在跨平台桌面应用领域，如果不考虑将 Web 技术打包成桌面应用（比如 Electron），Flutter 基本上已经是天花板级的选择了。如果连 Flutter 都无法满足需求，那通常就不会再追求跨平台方案，而是针对每个平台分别进行原生开发。</dd><br/>

以及，从另一位 Mac 应用开发者那里了解到：

<dd>当年 Swift 刚发布时，他满怀热情地冲去学习，写了不少代码。可没想到这门语言更新起来“大刀阔斧”，不到一年语法就大变样，老代码直接编译不过，只好大面积重写。每次更新还是免不了踩坑，维护成本高，感觉像在“一年学一次新语法”。</dd><br/>

<dd>最终他实在扛不住这种循环，彻底放弃了手写 Swift。现在他的策略很直接：让 AI 来生成代码，需要时直接用。</dd><br/>

结合以上信息，我已经知道了自己应该使用了哪些技术栈和工具来实现跨平台桌面应用的开发，并成功应用了在项目的实践中。
