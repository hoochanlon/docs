## 结论前置

这套符号体系高度依赖特定平台的渲染支持，一旦切换平台便会失效。这本质上是文档工具的一种"软性锁定"，通过培养用户的表达习惯来增加迁移成本。因此，最好将其整理为一份独立的备忘手册，以便在不同环境中随时取用。

:::tip
**重要提示**：本文档中的所有组件示例都需要在 `.mdx` 文件中使用，`.md` 文件不支持 JSX 组件。所有 `import` 语句必须放在文件顶部。
:::

## CSS 选择器精确匹配技巧

### 如何只选择侧边栏中的 Badge？

如果你只想针对侧边栏中的 `sl-badge` 设置样式，而不影响页面内容中的 badge，需要使用**后代选择器**来限定范围：

```css
/* ✅ 方法 1：使用侧边栏 ID（最推荐，最稳定） */
#starlight__sidebar .sl-badge {
    font-size: 10px !important;
}

/* ✅ 方法 2：使用侧边栏类名 */
.sl-sidebar .sl-badge {
    font-size: 10px !important;
}

/* ✅ 方法 3：使用导航属性 */
nav[aria-label="Main"] .sl-badge {
    font-size: 10px !important;
}
```

**选择器优先级说明：**
- `.sl-badge` - 会影响所有 badge（包括侧边栏和内容区）
- `#starlight__sidebar .sl-badge` - 只影响侧边栏内的 badge（ID 选择器，最稳定）
- `.sl-sidebar .sl-badge` - 只影响侧边栏内的 badge（类选择器）
- `nav[aria-label="Main"] .sl-badge` - 只影响主导航内的 badge（属性选择器）

### 其他精确选择示例

```css
/* 只选择侧边栏中的链接 - 使用 ID（最稳定） */
#starlight__sidebar a {
    font-size: 16px;
}

/* 只选择侧边栏中的链接 - 使用类名 */
.sl-sidebar a {
    font-size: 16px;
}

/* 只选择内容区域的标题 */
.sl-content h1 {
    font-size: 24px;
}

/* 只选择头部中的按钮 */
.sl-header button {
    /* 样式 */
}

/* 只选择特定容器内的元素 */
.sl-content .sl-badge {
    /* 只影响内容区的 badge，不影响侧边栏 */
}
```

### Starlight 常用 ID 选择器

Starlight 提供了一些稳定的 ID 选择器，优先使用这些：

```css
/* 侧边栏 */
#starlight__sidebar {
    /* 样式 */
}

/* 目录 */
#starlight__on-this-page {
    /* 样式 */
}

/* 使用示例 */
#starlight__sidebar .sl-badge {
    font-size: 10px;
}

#starlight__sidebar a {
    font-size: 16px;
}
```

### 选择器优先级规则

CSS 选择器的优先级（从低到高）：
1. 标签选择器：`div`、`a`、`h1`
2. 类选择器：`.sl-badge`、`.sl-sidebar`
3. ID 选择器：`#starlight__on-this-page`
4. 属性选择器：`[data-open-modal]`
5. 组合选择器：`.sl-sidebar .sl-badge`（更具体 = 更高优先级）

**原则**：选择器越具体，优先级越高，越不容易被覆盖。

## 2. 徽章（Badges）

徽章用于标记状态、版本或重要标签。

### 代码示例

```mdx
import { Badge } from '@astrojs/starlight/components'

- <Badge text="新功能" variant="tip" size="small" />
- <Badge text="已弃用" variant="caution" size="medium" />
- <Badge text="演示" variant="note" size="large" />
- <Badge text="200" variant="success" />
- <Badge text="实验性" variant="danger" />
```

### 参数说明

- `text`: 徽章显示的文本
- `variant`: 样式变体，可选值：`tip`、`caution`、`note`、`success`、`danger`
- `size`: 尺寸，可选值：`small`、`medium`、`large`

### 实际效果

- <Badge text="新功能" variant="tip" size="small" />
- <Badge text="已弃用" variant="caution" size="medium" />
- <Badge text="演示" variant="note" size="large" />
- <Badge text="200" variant="success" />
- <Badge text="实验性" variant="danger" />

## 注意事项

1. **文件格式**：所有使用组件的文件必须是 `.mdx` 格式，不能是 `.md` 格式
2. **导入位置**：所有 `import` 语句必须放在文件顶部，frontmatter 之后
3. **标签闭合**：所有 JSX 标签必须正确闭合，例如 `</LinkButton>` 不能写成 `</LinkButton`
4. **组件嵌套**：组件可以嵌套使用，但要注意正确的结构
5. **Markdown 兼容**：组件内部可以使用标准的 Markdown 语法
